Core Action Schemas

JSON Schema definitions for all SoEasily core actions

Triggers

Action Description Schema
Manual Trigger
Starts a workflow manually from the SoEasily UI. Can be configured with a custom JSON Schema to present a form for providing data when the workflow is run.
Schedule Trigger
Automatically starts a workflow based on a CRON schedule. Supports complex scheduling patterns and timezone configurations.
Webhook Trigger
Starts a workflow when an HTTP request is received at a specific endpoint. Supports various HTTP methods and authentication options.

Control Flow

Action Description Schema
Conditional
Routes the workflow down different branches based on a condition. Supports complex logical expressions and data comparisons.
Switch
Routes the workflow down different branches based on the value of a specified input field. Define routing rules in the outgoing connections.
Loop
Repeats a sequence of actions for each item in an array or until a condition is met. Supports iteration control and data aggregation.
Merge Branches
Combines multiple execution paths back into one. Only passes through the data from the single branch that was executed.
Wait State
Pauses workflow execution for a specified duration or until a specific time. Supports various delay formats and scheduling options.
Execute Workflow
Executes another workflow (a 'sub-workflow') using the input data provided. The current workflow waits for completion and receives the output.
Stop Workflow
Immediately halts the execution of the workflow and marks its final status as 'succeeded' or 'failed'.
Error Handler
Catches and handles errors from previous actions in the workflow. Provides error recovery and alternative execution paths.

Data Operations

Action Description Schema
Set Data
Creates or modifies data objects by setting specific fields to static values or expressions. Useful for data initialization and transformation.
Transform Data
Transforms input data using JSONata expressions. Supports complex data manipulation, filtering, and restructuring operations.
Query Data
Extracts specific data from the input using JSONata queries. Perfect for filtering, selecting, and extracting subsets of data.
Convert Data
Converts data between different formats (JSON, XML, CSV, etc.). Supports various encoding and transformation options.

Execution

Action Description Schema
Execute Code
Executes custom JavaScript code in a sandboxed environment. Provides access to input data and allows custom logic implementation.
Execute Command
Executes shell commands and system operations. Captures output, handles errors, and supports various execution environments.

Integration

Action Description Schema
HTTP Request
Makes HTTP requests to external APIs and services. Supports all HTTP methods, authentication, headers, and response handling.
GraphQL Request
Executes GraphQL queries and mutations against GraphQL endpoints. Supports variables, fragments, and error handling.

AI & LLM

Action Description Schema
LLM Prompt
Sends prompts to Large Language Models and retrieves generated text responses. Supports various AI providers and model configurations.
Agent Executor
Executes a sequence of actions driven by a Large Language Model to achieve a high-level objective. The agent has access to defined tools.

Utilities

Action Description Schema
Debug Log
Logs data for debugging and monitoring purposes. Outputs structured log entries with configurable levels and formatting.
No Operation
Passes input data through unchanged. Useful for testing, placeholders, and workflow structure during development.
