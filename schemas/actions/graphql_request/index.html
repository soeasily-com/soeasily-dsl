GraphQL Request

Sends a query or mutation to a GraphQL endpoint and returns the response. Perfect for modern API integrations with precise data fetching.

Overview

The GraphQL Request action enables seamless integration with GraphQL APIs, offering precise data fetching with queries and data modification with mutations. Unlike REST APIs that require multiple endpoints, GraphQL allows you to request exactly the data you need in a single request, making it perfect for efficient API integrations and reducing over-fetching of data.

GraphQL vs REST

Understanding the key differences between GraphQL and REST API approaches.

Aspect GraphQL REST
Data Fetching Request exactly what you need Fixed data structure per endpoint
Endpoints Single endpoint for all operations Multiple endpoints for different resources
Over/Under-fetching Eliminated by design Common issue with fixed responses
Versioning Schema evolution without versioning API versioning required for changes
Real-time Built-in subscriptions support Requires additional technologies
Caching More complex due to dynamic queries Simple HTTP caching

GraphQL Examples

See how GraphQL Request handles different types of queries and mutations.

Simple Query

Fetch specific country information

Configuration:

{
  "endpoint": "https://countries.trevorblades.com/",
  "query": "query { country(code: \"US\") { name capital currency } }"
}

GraphQL Query:

query { country(code: "US") { name capital currency } }

Response:

{ "status_code": 200, "data": { "country": { "name": "United States", "capital": "Washington, D.C.", "currency": "USD" } } }

Query with Variables

Use variables for dynamic queries

Configuration:

{
  "endpoint": "https://api.github.com/graphql",
  "query": "query($owner: String!, $name: String!) { repository(owner: $owner, name: $name) { name description stargazerCount } }",
  "variables": {
    "owner": "facebook",
    "name": "react"
  },
  "headers": {
    "Authorization": "Bearer YOUR_TOKEN"
  }
}

GraphQL Query:

query($owner: String!, $name: String!) { repository(owner: $owner, name: $name) { name description stargazerCount } }

Variables:

{ "owner": "facebook", "name": "react" }

Mutation Example

Create or update data with mutations

Configuration:

{
  "endpoint": "https://api.example.com/graphql",
  "query": "mutation($input: CreateUserInput!) { createUser(input: $input) { id name email } }",
  "variables": {
    "input": {
      "name": "John Doe",
      "email": "john@example.com"
    }
  }
}

GraphQL Mutation:

mutation($input: CreateUserInput!) { createUser(input: $input) { id name email } }

Complex Nested Query

Fetch related data in a single request

GraphQL Query:

query { user(id: "123") { name email posts { title content comments { author text createdAt } } followers { name avatar } } }

Benefit: This single request replaces what would require multiple REST API calls (user details, posts, comments, followers).

Common GraphQL Patterns

Useful patterns for different GraphQL operations and use cases.

Basic Query

# Simple field selection
query {
  user(id: "123") {
    name
    email
  }
}

# With arguments
query {
  posts(limit: 10, offset: 0) {
    title
    author
  }
}

Query with Variables

# Define variables
query GetUser($userId: ID!, $includeEmail: Boolean!) {
  user(id: $userId) {
    name
    email @include(if: $includeEmail)
  }
}

# Variables object
{
  "userId": "123",
  "includeEmail": true
}

Mutations

# Create operation
mutation CreatePost($input: PostInput!) {
  createPost(input: $input) {
    id
    title
    createdAt
  }
}

# Update operation
mutation UpdateUser($id: ID!, $data: UserUpdateInput!) {
  updateUser(id: $id, data: $data) {
    id
    name
    updatedAt
  }
}

Fragments

# Define reusable fragments
fragment UserInfo on User {
  id
  name
  email
  avatar
}

# Use in queries
query {
  currentUser {
    ...UserInfo
  }
  friends {
    ...UserInfo
  }
}

Aliases

# Use aliases for multiple calls
query {
  firstUser: user(id: "1") {
    name
  }
  secondUser: user(id: "2") {
    name
  }
}

Pagination

# Cursor-based pagination
query GetPosts($first: Int!, $after: String) {
  posts(first: $first, after: $after) {
    edges {
      node {
        id
        title
      }
      cursor
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}
