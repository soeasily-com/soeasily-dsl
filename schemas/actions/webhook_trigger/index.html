Webhook Trigger

Creates a unique URL to start the workflow when an HTTP request is received. Captures the request method, headers, query parameters, and body to be used in subsequent actions.

Overview

The Webhook Trigger creates a unique HTTP endpoint that external systems can call to start your workflow. When a request is received, it captures all the request details (method, headers, query parameters, and body) and makes them available to subsequent actions. Perfect for integrating with third-party services, receiving notifications, or building API endpoints.

How It Works

When you add a Webhook Trigger to your workflow, SoEasily automatically generates a unique URL for that workflow. External systems can send HTTP requests to this URL to trigger workflow execution.

https://webhooks.soeasily.com/trigger/wf_abc123def456

1. External Request

An external system sends an HTTP request to your webhook URL with data.

2. Request Capture

SoEasily captures all request details: method, headers, query params, and body.

3. Workflow Execution

Your workflow starts with the captured request data available to all actions.

Security & Authentication

Protect your webhook endpoints with optional authentication mechanisms.

No Authentication

Configuration: "authentication": "none"
Open endpoint that accepts any request. Use for public webhooks or internal systems.

Token Authentication

Configuration: "authentication": "token"
Requires a valid token in the Authorization header. Provides basic security for sensitive workflows.
