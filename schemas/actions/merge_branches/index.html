Merge Branches

Combines multiple execution paths back into one. Only passes through the data from the single branch that was executed.

Overview

The Merge Branches action is the essential counterpart to conditional branching in workflows. After your workflow splits into multiple paths using Conditional or Switch actions, Merge Branches brings those paths back together into a single execution flow. It acts as a convergence point where different branches reunite, ensuring your workflow continues with a unified path regardless of which branch was taken.

🔀 Path Convergence

Brings multiple workflow branches back together into a single execution path.

📊 Data Preservation

Passes through data from whichever branch was actually executed, maintaining data integrity.

🎯 Clean Architecture

Creates clean, readable workflow structures by clearly defining where branches end.

How Merge Branches Works

Merge Branches waits for input from one of its connected branches and passes that data through unchanged.

Common Workflow Patterns

Typical patterns where Merge Branches is used to reunify workflow execution paths.

If-Then-Else Pattern

Conditional → Branch A / Branch B → Merge → Continue

Classic conditional branching where different actions are taken based on conditions, then merged back together.

Switch-Case Pattern

Switch → Multiple Branches → Merge → Continue

Multi-way branching where different paths are taken based on values, then unified at the merge point.

Error Handling Pattern

Try Action → Success / Error Handler → Merge → Continue

Error handling where successful execution and error recovery paths merge back together.

Parallel Processing Pattern

Split → Process A / Process B → Merge → Combine Results

When different processing paths need to be reunified for final processing or output.

Best Practices

🎯 Clear Structure

Design clear branching structures:
• Use Merge Branches for every branching point
• Keep branch logic simple and focused
• Document the purpose of each branch
• Ensure all branches lead to the merge point

📊 Data Consistency

Maintain consistent data structures:
• Ensure all branches output compatible data
• Use consistent field names across branches
• Handle missing fields appropriately
• Document expected data formats

🔍 Testing

Test all branch paths:
• Verify each branch reaches the merge point
• Test with different input scenarios
• Validate data integrity through merges
• Check error handling paths

📝 Documentation

Document workflow logic:
• Explain branching conditions clearly
• Document what each branch does
• Describe merge point expectations
• Include example data flows
