Execute Workflow

Executes another workflow (a 'sub-workflow') using the input data provided. The current workflow will wait for the sub-workflow to complete and will receive its final output.

Overview

The Execute Workflow action enables modular workflow design by allowing you to call other workflows as sub-processes. This powerful capability promotes reusability, maintainability, and clean architecture by breaking complex processes into smaller, focused workflows. Perfect for creating workflow libraries, implementing common business processes, and building scalable automation systems.
