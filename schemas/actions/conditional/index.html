Conditional

Allows for simple 'if/then' branching in a workflow. The input data is passed through, and logic is applied to the outgoing connections' 'condition' fields to determine the execution path.

Overview

The Conditional action is the foundation of intelligent workflow routing. It acts as a decision point that evaluates data and directs workflow execution down different paths based on conditions. Unlike complex branching logic, the Conditional action keeps things simple by passing data through unchanged while letting you define conditions on the connections themselves.

🔀 Smart Routing

Route workflow execution based on data values, user properties, or any conditional logic.

📊 Data Preservation

Passes input data through unchanged, maintaining data integrity across branches.

⚡ Zero Configuration

No setup required - all logic is defined on the outgoing connections for maximum flexibility.

Tags

if else branch condition logic route

How It Works

The Conditional action works by evaluating conditions defined on its outgoing connections. When data flows into the action, it's passed through unchanged, but the workflow engine evaluates each connection's condition to determine which path(s) to follow.

Example: User Age Verification

{
  "connections": [
    {
      "source_action": "conditional_1",
      "target_action": "send_adult_content",
      "condition": "user.age >= 18",
      "data_mapping": {
        "user_data": "source.user"
      }
    },
    {
      "source_action": "conditional_1", 
      "target_action": "request_parental_consent",
      "condition": "user.age < 18",
      "data_mapping": {
        "user_data": "source.user"
      }
    }
  ]
}

Example: Order Value Routing

{
  "connections": [
    {
      "source_action": "conditional_1",
      "target_action": "auto_approve",
      "condition": "order.value <= 100",
      "data_mapping": {
        "order": "source.order"
      }
    },
    {
      "source_action": "conditional_1",
      "target_action": "manual_review",
      "condition": "order.value > 100",
      "data_mapping": {
        "order": "source.order"
      }
    }
  ]
}

Common Use Cases

👤 User Segmentation

Route users based on subscription level, location, or account status for personalized experiences.

💰 Order Processing

Direct orders to different approval processes based on value, customer type, or payment method.

🎯 Content Filtering

Show different content or features based on user permissions, age, or preferences.

⚠️ Error Handling

Route to different error handling paths based on error type or severity level.

📊 Data Validation

Send valid data to processing and invalid data to correction or notification flows.

🕐 Time-based Logic

Route workflows differently based on time of day, day of week, or business hours.

Condition Expressions

Conditions are evaluated using JMESPath or JSONata expressions. Here are common patterns:

Numeric Comparisons

user.age >= 18
order.total > 100
score <= 85

String Matching

user.status == 'premium'
country == 'US'
email.endsWith('@company.com')

Boolean Logic

user.verified && user.active
status == 'pending' || status == 'review'
!user.blocked

Array Operations

tags.includes('urgent')
permissions.length > 0
roles.contains('admin')
