Stop Workflow

Immediately halts the execution of the workflow and marks its final status as 'succeeded' or 'failed'. Provides controlled termination for conditional workflow completion.

Overview

The Stop Workflow action provides controlled termination of workflow execution, allowing you to explicitly end a workflow with a specific status and message. Unlike natural workflow completion, Stop Workflow gives you precise control over when and how a workflow ends, making it perfect for conditional termination, early exits, and explicit success or failure scenarios.

🛑 Controlled Termination

Explicitly stop workflow execution at any point with full control over the final status.

📊 Status Control

Set the workflow's final status as either 'succeeded' or 'failed' with optional messages.

🎯 Conditional Exits

Enable early workflow termination based on business logic and conditions.

Tags

stop end terminate halt exit

How Stop Workflow Works

Stop Workflow immediately terminates the current workflow execution and sets the final status.

SUCCEEDED
Workflow completed successfully
FAILED
Workflow terminated with failure
1. Workflow reaches Stop Workflow action
2. Execution immediately halts
3. Final status is set (succeeded/failed)
4. Optional message is recorded
5. Workflow execution ends

Stop Examples

See how Stop Workflow provides controlled termination in different scenarios.

Conditional Success Termination

Stop workflow early when conditions are already met

Configuration:

{
  "status": "succeeded",
  "message": "User already exists, no action needed"
}

Workflow Context:

Check if user exists → User found → Stop Workflow (succeeded)

Use Case:

In a user creation workflow, if the user already exists, there's no need to continue with the creation process. The workflow can stop successfully since the desired end state (user exists) is already achieved.

Validation Failure Termination

Stop workflow when critical validation fails

Configuration:

{
  "status": "failed",
  "message": "Invalid email format provided, cannot proceed with user registration"
}

Workflow Context:

Validate email format → Invalid format detected → Stop Workflow (failed)

Use Case:

When processing user registration, if the email format is invalid, continuing the workflow would be pointless and potentially harmful. Stop immediately with a clear failure message.

Business Rule Enforcement

Stop workflow when business rules prevent continuation

Configuration:

{
  "status": "failed",
  "message": "Order amount exceeds customer credit limit ($5,000). Manual approval required."
}

Workflow Context:

Check credit limit → Amount exceeds limit → Stop Workflow (failed)

Use Case:

In an order processing workflow, if the order amount exceeds the customer's credit limit, the automated workflow should stop and require manual intervention.

Resource Availability Check

Stop workflow when required resources are unavailable

Configuration:

{
  "status": "failed",
  "message": "External API service is currently unavailable. Workflow cannot proceed."
}

Workflow Context:

Check API availability → Service down → Stop Workflow (failed)

Early Success Completion

Stop workflow when objective is achieved early

Configuration:

{
  "status": "succeeded",
  "message": "Data synchronization already up to date, no changes needed"
}

Workflow Context:

Compare data timestamps → Already synchronized → Stop Workflow (succeeded)

Configuration Schema

Configure Stop Workflow with the desired final status and optional termination message. 

{
  "title": "Stop Workflow Configuration",
  "type": "object",
  "properties": {
    "status": {
      "type": "string",
      "enum": ["succeeded", "failed"],
      "description": "The final status to assign to the workflow run."
    },
    "message": {
      "type": "string",
      "description": "An optional message to record as the reason for stopping."
    }
  },
  "required": ["status"]
}

Common Workflow Patterns

Typical patterns where Stop Workflow provides controlled termination.

Guard Clauses

Early termination based on preconditions:
• Check prerequisites before processing
• Validate input data quality
• Verify resource availability
• Enforce business rules

Conditional Success

Stop when objectives are already met:
• Data already synchronized
• User already exists
• Process already completed
• No action required

Error Boundaries

Controlled failure handling:
• Critical validation failures
• Resource unavailability
• Security violations
• Business rule violations

Circuit Breakers

Prevent cascading failures:
• External service failures
• Rate limit exceeded
• System overload conditions
• Dependency failures

Common Use Cases

✅ Validation Gates

Stop workflows when critical validation fails, preventing invalid data processing.

🛡️ Security Enforcement

Terminate workflows when security violations or unauthorized access is detected.

📊 Business Rules

Enforce business logic by stopping workflows when rules are violated.

🔄 Idempotency

Stop successfully when operations are already completed to ensure idempotent behavior.

🚨 Emergency Stops

Provide emergency termination when critical errors or dangerous conditions are detected.

⚡ Resource Management

Stop workflows when required resources are unavailable or limits are exceeded.

Best Practices

📝 Clear Messages

Provide informative termination messages:
• Explain why the workflow stopped
• Include relevant context and data
• Suggest next steps when appropriate
• Use consistent message formats

🎯 Appropriate Status

Choose the correct status for each scenario:
• Use 'succeeded' when objectives are met
• Use 'failed' for errors and violations
• Consider business context, not just technical success
• Document status decision criteria

🔍 Early Placement

Place Stop Workflow actions strategically:
• Check critical conditions early
• Validate before expensive operations
• Use after conditional branches
• Position for maximum efficiency

📊 Monitoring

Monitor workflow terminations:
• Track stop reasons and frequency
• Alert on unexpected failure patterns
• Analyze termination messages
• Use data to improve workflows
