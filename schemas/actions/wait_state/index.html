Wait

Pauses the workflow for a specified duration before proceeding. Useful for delays, rate limiting, or waiting for external processes.

Overview

The Wait action introduces controlled delays into your workflows. Whether you need to respect API rate limits, allow time for external processes to complete, implement retry delays, or simply pace your workflow execution, the Wait action provides precise timing control while preserving your data flow.
