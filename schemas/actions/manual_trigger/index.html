Manual Trigger

Starts a workflow manually from the SoEasily UI. This trigger can be configured with a custom JSON Schema to present a form for providing data when the workflow is run.

Overview

The Manual Trigger is the simplest way to start a workflow. It allows users to manually initiate workflow execution from the SoEasily interface, optionally collecting input data through a customizable form. This makes it perfect for testing workflows, running one-off processes, or creating user-initiated automations.

Configuration Schema

Configure the Manual Trigger with an optional input form schema that defines what data users can provide when starting the workflow. 

{
  "title": "Manual Trigger Configuration",
  "type": "object",
  "properties": {
    "payload_schema": {
      "title": "Input Form Schema",
      "description": "A JSON Schema defining the input form presented to the user at runtime. The data provided will become the output of this trigger.",
      "type": "object",
      "default": {
        "type": "object",
        "properties": {}
      }
    }
  },
  "additionalProperties": false
}

Input & Output

Input Schema

This trigger does not accept input from other actions as it's designed to start workflows.

{
  "title": "Manual Trigger Input",
  "description": "This trigger does not accept input from other actions.",
  "type": "object",
  "properties": {},
  "additionalProperties": false
}

Output Schema

The output is dynamic and conforms to the structure defined in the 'Input Form Schema' configuration.

{
  "title": "Manual Trigger Output",
  "description": "The output is dynamic and conforms to the structure defined in the 'Input Form Schema' configuration.",
  "type": "object"
}

Run with a Simple Text Input

Simulates a user providing a customer name in a custom form.

Configuration:

{
  "payload_schema": {
    "type": "object",
    "title": "New User Form",
    "properties": {
      "customer_name": { 
        "type": "string", 
        "title": "Customer Name" 
      }
    },
    "required": ["customer_name"]
  }
}

Expected Output:

{
  "customer_name": "Jane Doe"
}

Run with Numeric and Boolean Inputs

Simulates a user providing an order ID and specifying if it's a priority order.

Configuration:

{
  "payload_schema": {
    "type": "object",
    "properties": {
      "order_id": { "type": "integer" },
      "is_priority": { 
        "type": "boolean", 
        "default": false 
      }
    }
  }
}

Expected Output:

{
  "order_id": 95821,
  "is_priority": true
}
